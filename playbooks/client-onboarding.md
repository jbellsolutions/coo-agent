# Client Onboarding Playbook

How the COO manages new client intake and vendor relationships.

---

## Client Onboarding Flow

```
INTAKE → SETUP → ACTIVATE → VERIFY → HANDOFF
```

### Phase 1: Intake (Day 0)
**Trigger:** Justin closes a deal or approves a new client.

**COO Actions (automated):**
1. Create ClickUp project from client template
2. Create GHL contact record with all known details
3. Create client folder in Google Drive
4. Create #client-[name] Slack channel
5. Log the deal value in revenue tracker
6. Set up billing milestone reminders in ClickUp

**Required Information:**
```
CLIENT INTAKE FORM
━━━━━━━━━━━━━━━━━━━━━━━━━
Company: ___________________________________
Contact: ___________________________________
Email:   ___________________________________
Phone:   ___________________________________
Deal Value: $_______ | Payment Terms: ________
Start Date: _________ | Deadline: ____________
Deliverables:
  1. ________________________________________
  2. ________________________________________
  3. ________________________________________
Special Requirements: ________________________
```

### Phase 2: Setup (Day 1-2)
**COO Actions:**
1. Verify payment/deposit received (Rule 29: No Build Before Payment)
2. Create project timeline with milestones in ClickUp
3. Set up project-specific monitoring (if applicable)
4. Share relevant Google Drive folder with client
5. Send welcome message template to Justin for approval

### Phase 3: Activate (Day 3+)
**COO Actions:**
1. Kick off first deliverable tasks
2. Set daily project status tracking
3. Monitor progress against timeline

### Phase 4: Verify (At Milestones)
**COO Actions:**
1. Run quality check on deliverables
2. Compile client update report
3. Track hours/costs against budget

### Phase 5: Handoff (Project Complete)
**COO Actions:**
1. Final deliverable quality audit
2. Compile project summary (timeline, costs, outcomes)
3. Update revenue tracker with final numbers
4. Archive project in ClickUp
5. Schedule follow-up reminder (30 days) for referral/upsell

## Vendor Evaluation Framework

When evaluating new tools or services, use this matrix:

### Evaluation Criteria (Score 1-5)

| Criteria | Weight | Questions to Answer |
|----------|--------|-------------------|
| **Functionality** | 25% | Does it solve the core problem? Feature completeness? |
| **Cost** | 20% | Monthly/annual cost? Cost per unit of value? Free tier? |
| **Integration** | 20% | Has MCP/API? Works with our stack? Setup complexity? |
| **Reliability** | 15% | Uptime SLA? Support responsiveness? Maturity? |
| **Scalability** | 10% | Can it grow with us? Pricing at 10x volume? |
| **Lock-in Risk** | 10% | Data export? Migration path? Contractual lock-in? |

### Vendor Review Schedule
- **Quarterly:** Review all active vendors for cost optimization
- **On trigger:** When any vendor has reliability issues, price increase, or feature gap
- **Annual:** Full stack audit — are we using the best tools for each job?
